# 面试题目

```markdown
1. 面试基本要求包括统计基础和SQL能力
2. 在分析场景时，用户需求、痛点和解决方案要一一对应，概念不能混淆
3. 产品培训项目竞争激烈，虽然能证明毕业生能力，但入职后离职率较高
4. 初面形式：一位产品经理同时面试两位候选人，主要讨论PPT内容并进行问答
5. 笔试内容包括行测题、开放题和性格测试
6. 数据分析是必考内容，包括费米估算问题
7. 请谈谈你对这个产品商业模式的看法
8. 请复盘你某个AI项目最大的失败点及改进方法
9. MAE和RMSE指标分别适用于什么场景？
10. 智力题：给出一组数字，预测下一个数字（考察数字敏感度）
11. SQL中不同join类型的区别是什么？
12. 在字节的经历中，某个策略是如何落地的？
13. 字节商业化业务的主要目标是什么？
14. 你最常用音乐软件的哪个功能？
15. 复盘时发现了哪些问题？你的方案是否有可推广性？
16. 模型复杂度指的是什么？
17. 你希望QQ音乐删除哪个功能？为什么？
18. 产品经理设计这个功能时考虑了哪些因素？
19. 请介绍马尔可夫模型
20. 这个项目的核心问题是什么？
21. 优化这个功能的关键指标是什么？
22. 为什么从技术转产品？考察职业规划
23. 请介绍字节商业化中做得好的项目
24. 为腾讯会议设计一个AI功能，说明应用场景和技术实现
25. 请介绍百度商业化中效果好的项目
26. AI在广告中有哪些应用？
27. 现场完成四道SQL题目
28. 当你接手新业务时，如何快速熟悉？
29. 特征工程有哪些常用方法？
30. 请介绍一段实习经历和你的角色
31. 如果产品ROI下降，如何分析原因？
32. 为什么选择儿童遗尿症作为第一个项目？
33. 指出QQ音乐的一个不足并提出改进方案
34. 你用过哪些AI产品？它们的优缺点是什么？
35. 如何处理数据中的缺失值？
36. 为什么选择这几家互联网公司实习？是因为个人偏好还是其他原因？
37. 如何看待小红书的推荐逻辑？为什么商业化推进困难？
38. 解释Transformer和CNN的区别。如何评估模型效果？
39. 不同行业客户在广告投放时关注哪些不同指标？
40. 你做数据分析的方法论是什么？
41. 作为产品经理，如何优化视频号广告？
42. 现场完成两道SQL题
43. 如果算法团队认为需求ROI低，如何说服他们？
44. 如何看待AIGC对腾讯内容生态的影响？
45. 你对AI大模型的理解是什么？
46. 遇到最大的团队矛盾时如何解决？
47. 这个产品的核心竞争优势是什么？
48. 你的技术栈有哪些？
49. 如何设计AI工具的收费模式（如按API调用或订阅制）？
50. 你平时用抖音还是快手？为什么选择去快手实习？
51. 你用过哪些爬虫工具？
52. AI中台如何平衡不同业务线的定制需求？
53. 请介绍你最近常用的产品
54. 如果QQ音乐日活变化，可能是什么原因？
55. 为什么想做产品经理？
56. 如何看待AI产品经理的发展前景？
57. 在字节实习的业务目标是什么？你负责什么？达成情况如何？
58. 作为商业化策略产品经理，你会关注哪些指标？
59. 为了达成目标，你采取了哪些措施？

66. XGBoost为什么效果好？
67. 工作中遇到困难时如何解决？
68. B端和C端AI产品的核心决策链有什么区别？
69. 刚才提到的效果提升，具体是怎么计算的？
70. 你最喜欢这个产品的哪个功能？为什么？
71. 如何处理AI伦理问题（如数据隐私与产品目标的冲突）？
72. 准备一段深度参与的实习经历，能回答90%以上的相关问题
73. 思考时可以先用AI搜索现有产品案例
74. 请详细介绍三个项目经历
75. 考察发现问题的能力和选题理由
76. 用STAR法则描述项目，突出技术落地
77. HR主要通过关键词筛选简历（如B端、策略、数据等）
78. 提前研究腾讯AI布局（如混元大模型）
79. 通过详细描述工作过程来展示能力，3-4个项目足够
80. 做社交内容产品时，要清楚业务指标并体现思考
81. 请提供作品集链接（含原型图和技术方案）
82. 需要逻辑清晰、记录快速
83. AIPM不是纯技术岗，要能用产品语言解释技术价值
84. 项目复盘情况？需要改进的地方？是否具备运营能力？
```

# 面试经验

```markdown
1. 腾讯面试不需要太紧张，做好充分准备记住重点就行
2. 本来因为缺乏运营和产品实习经验，没抱希望进大厂暑期实习，只想来锻炼一下，没想到进了二面
3. 腾讯的面试风格很特别，不同水平的候选人会在同一个群面里。表达清晰逻辑好的才能进入下一轮，简历上的每个项目都会被深挖
4. 不太喜欢面试部门的业务，而且小组氛围太激进了
5. 每次发言前要先想好怎么说得更有逻辑和效率。第一次回答时不要急着说，应该想完整了再回答
6. 不管适不适合产培项目，都建议投递试试，当作秋招前的锻炼，但要保持轻松心态
7. 这次面试让我意识到之前太关注B端产品的业务和功能设计，忽略了商业化部分，以后要多关注
8. 这次经历让我相信认真准备实习和面试是必要的
9. 面试到最后已经没什么精神，不太想说话了
10. 我是第一个发言的。后面同学都说前面说得比较完整了，他们只做了一些补充
11. 我们最后的设计方案缺乏结构化框架
12. 没想到第一个offer是腾讯给的
13. 这次面试是很宝贵的经历。第一次参加大厂群面，虽然不记得什么时候投的。各种体验都很新鲜，确实能找到需要改进的地方
14. 群面时我们组气氛很好，像小组讨论，没有明确分工。感觉只要积极参与讨论，适时总结要点就行
15. 从一面到云证大概要半个月
16. 面试官很和蔼，但我总觉得他笑里藏刀
17. 感谢面试官真诚给建议，虽然没通过但体验不错
18. 一面聊得很愉快。一般是同事或mentor面。面试官很友善，问了简历和业务问题，没问竞品分析这类泛问题。最后简单问了几个BQ。晚上刷新显示通过
19. 我们组只有3个人，和攻略上说的不一样，没有抢话情况，大家都很礼貌，我运气不错
20. HR面/测评每年形式不同，通过的要线下集合做测评和团队游戏，比较考验体力，HR在旁边观察综合素质
21. 产培有个特色环节是小组根据业务热点出方案。每组通过0-2人，通过后当晚要做PPT
22. 虽然岗位要求技术背景，但面试还是考察过往经历和产品感觉
23. 这是总监面
24. 我无意中当了timer，努力提醒时间但效果不好。最后才选汇报代表，大家积极性不高。我又主动承担了任务
25. 第一次群面感觉一般
26. 每天都在为系统填报、投简历、做测评和等回复焦虑
27. 从投递到收到offer，三个事业部联系过我
28. 面试流程：组长面、交叉面、总监面、GM面、HR面
29. 没有明确timer时不要勉强当，因为大家都在想方案不想被打扰
30. 结束时我问了面试官对AI Agent在C端爆发节点的看法，对方认真回答了5分钟。这种主动思考可能是加分项
31. 总共面了7轮
32. 你最想聊哪段互联网实习？
33. 你做策略产品经理的优劣势是什么？
34. 介绍在百度商业化遇到的失败经历
35. 你认为失败的原因是什么？
36. 好的产品经理简历要讲好三个故事
37. 自由讨论时如果梳理到核心功能点，可以把之前内容总结成几个要点
38. 不要生搬硬套框架
39. 保持轻松心态
40. 可以赞同他人观点并拓展，或指出潜在风险
41. 放平心态等待机会。鹅厂像复活赛，失败可以等机会，遇到不适合的业务也可以拒绝。面试也是认识自己的过程
42. 业务面和HR面最后都会问个人优缺点、兴趣爱好等。回顾自己的经历，真实表现比套路回答更有深度
43. 腾讯看重思维能力。没有学历歧视，接受零实习的。更看重逻辑和思考深度。相比硬指标，提供了更多弥补机会
44. 向HR说明经验匹配情况
45. 向面试官展示落地能力
46. 你的规划能力、判断力和在互联网市场的认可度
47. 工作经历要包含做了什么、怎么做、结果如何三部分
48. 简历要针对面试官需求，不是罗列所有经历
49. 复盘问题总结经验供其他业务参考。考察复盘习惯和成果应用能力
50. 简历是门票，产品经理要会讲故事
51. 面试中要展现积极形象，面试官是唯一评判者。印象主要取决于整体表现
52. 告诉面试官自己能胜任
53. 准备充分就没问题
54. 简历要包含HR能识别的关键词
55. 一开始就要和recorder确认，这样更有权威性
56. 知道简历要讲的三个好故事，主线就自然形成了
57. 遇到压力问题先拆解本质，比如把"需求不合理"转化为关注用户痛点和数据验证
58. 掌握方法论就行
59. 简历排版要简洁专业，避免花哨
60. 面试是否继续由面试官决定，要在20秒内让面试官了解每段经历的项目内容和成果数据
61. 找到自己的记忆点，比如擅长AI+社交场景或有从零到一的AI商业化经验
62. 大家轮流发言时要记笔记
63. 关键是要在短时间内让人喜欢你
```
